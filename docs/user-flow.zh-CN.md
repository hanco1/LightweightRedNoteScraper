# 用户流程

## 主流程

1. 用户在手机浏览器里打开页面。
2. 粘贴一条公开的小红书 / RedNote 链接，或者直接粘贴分享文案。
3. 点击主按钮开始整理。
4. 前端将输入内容发送到 `POST /api/capture`。
5. 后端提取公开链接，访问公开页面，读取页面里的笔记状态，并整理成统一结果。
6. 页面展示：
   - 标题
   - 去标签后的正文
   - 标签
   - 发布时间
   - IP 归属地（如果公开数据里有）
   - 互动数据
   - 媒体卡片
7. 用户可以：
   - 查看原帖
   - 单独保存某个媒体
   - 一键保存全部媒体
   - 在预览异常时重新加载图片
8. 页面刷新后，当前结果直接清空，不保留历史。

## 为什么它是轻量的

- 只有一个主页面
- 单次抓取只走一次核心接口
- 不登录
- 不保存 cookies
- 不保留历史记录
- 没有额外后台和广告模块
