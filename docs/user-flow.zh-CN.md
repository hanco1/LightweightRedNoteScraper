# 用户流程

这是一个面向手机浏览器的公开小红书 / RedNote 采集工具。目标很直接：粘贴一条公开链接，抓取一次，把结果整理得清楚、可预览、可保存。

## 主要流程

1. 在手机浏览器中打开页面。
2. 粘贴公开的小红书 / RedNote 链接，或者包含该链接的分享文案。
3. 点击主按钮开始采集。
4. 前端向 `POST /api/capture` 发送输入内容。
5. 接口校验请求，提取公开 URL，抓取公开页面，解析内嵌的笔记状态，并整理成统一的数据结构。
6. 页面展示整理后的内容：
   - 标题
   - 去掉标签后的正文
   - 作者
   - 发布时间
   - IP 归属地（如果公开数据里有）
   - 点赞、收藏、评论、分享
   - 图片、视频和 Live Photo 素材卡片
7. 图片和视频预览、下载都通过同源代理 `/api/media` 完成，这样在移动浏览器里更稳定，Safari 兼容性也更好。
8. 用户可以打开原帖、单独保存某一项、一键保存全部，或者在预览失败时重新加载。
9. 刷新页面会清空当前结果；服务端不保留历史记录。

## 为什么它很轻量

- 只有一个页面和一条抓取路径
- 不登录，不使用 cookies
- 不建数据库，也不保存服务端历史
- 没有后台任务或管理面板
- 只有在需要预览或保存媒体时才走同源代理
