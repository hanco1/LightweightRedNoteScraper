# 架构说明

## 总览

这是一个面向公开笔记的轻量级小红书 / RedNote 抓取器。

整体围绕一个手机页面和两个无状态接口展开：

- `POST /api/capture`
- `GET /api/media`

## 运行结构

### 1. 手机前端页面

文件：

- `index.html`
- `styles.css`
- `app.js`
- `lib/i18n.js`

职责：

- 渲染手机端 UI
- 处理中英文切换
- 提交公开链接
- 展示抓取结果
- 处理单个保存与一键保存

### 2. 抓取接口

文件：

- `api/capture.js`

职责：

- 校验请求
- 从分享文本中提取公开链接
- 调用 `lib/xhs.js` 完成抓取
- 返回统一结构的 JSON 数据

### 3. 解析与标准化

文件：

- `lib/xhs.js`

职责：

- 识别公开笔记链接
- 获取公开 HTML 页面
- 提取 `window.__INITIAL_STATE__`
- 安全解析笔记状态
- 标准化标题、正文、标签、互动数、发布时间、IP 归属地和媒体
- 将原始媒体地址转换为可代理的同域地址

### 4. 媒体代理

文件：

- `api/media.js`

职责：

- 通过同域代理图片和视频
- 为预览和下载提供统一入口
- 转发视频播放所需的 Range 请求
- 提高 iPhone / Safari 下的兼容性

### 5. 本地开发服务

文件：

- `dev-server.mjs`

职责：

- 在本地提供与线上一致的页面和接口
- 代理 `/api/capture` 与 `/api/media`
- 方便调试和回归测试

## 数据流

1. 用户贴入公开链接。
2. 前端调用 `POST /api/capture`。
3. `api/capture.js` 调用 `fetchCaptureFromPublicLink`。
4. `lib/xhs.js` 获取公开页面并解析笔记状态。
5. 接口返回标准化后的 JSON。
6. 页面渲染标题、正文、标签、互动数据和媒体卡片。
7. 预览与保存操作通过 `/api/media?url=...` 完成。

## 为什么运行快

- 无数据库
- 无登录流程
- 无服务端历史记录
- 无后台任务
- 每次交互只处理一条公开笔记
- 只有预览和下载时才经过媒体代理
