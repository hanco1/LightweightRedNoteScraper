# 架构说明

这是一个面向公开小红书 / RedNote 内容的移动端优先采集器。它的设计目标是保持无状态：用户粘贴一条公开链接，应用抓取一次，服务端不保存历史。

## 运行结构

### 手机前端

文件：

- `index.html`
- `styles.css`
- `app.js`
- `lib/i18n.js`

职责：

- 渲染移动端界面
- 处理中英文文案
- 提交一条公开链接或分享文案
- 渲染整理后的结果
- 提供单独保存、全部保存和预览操作

### 抓取接口

文件：

- `api/capture.js`

职责：

- 接收 `POST` 请求
- 读取粘贴内容
- 调用 `fetchCaptureFromPublicLink`
- 返回整理后的 JSON，或者清晰的错误信息

### 解析与标准化

文件：

- `lib/xhs.js`

职责：

- 从粘贴文本中提取第一个公开的小红书 / RedNote 链接
- 使用浏览器风格请求头抓取公开页面
- 安全解析 `window.__INITIAL_STATE__`
- 标准化标题、正文、标签、作者、发布时间、IP 归属地、互动数据和媒体
- 将上游媒体地址转换为同源代理地址
- 只允许可信的小红书 / XHS 媒体域名进入代理层

### 媒体代理

文件：

- `api/media.js`

职责：

- 处理 `GET` 和 `HEAD` 请求
- 校验目标地址
- 转发 `Range` 请求以支持播放
- 通过同源方式流式转发媒体
- 提高手机浏览器里的图片和视频预览兼容性

### 本地开发

文件：

- `dev-server.mjs`

职责：

- 在本地提供与线上一致的前端
- 路由 `/api/capture` 和 `/api/media`
- 让本地调试尽量贴近生产环境

## 数据流

1. 用户粘贴公开链接或分享文案。
2. 前端将内容发送到 `POST /api/capture`。
3. `api/capture.js` 校验请求，并调用 `fetchCaptureFromPublicLink`。
4. `lib/xhs.js` 提取公开 URL，抓取公开页面，解析内嵌的笔记状态。
5. 抓取流程返回标准化 JSON，包含标题、正文、标签、作者、发布时间、IP 归属地、互动数据、来源链接、笔记类型和媒体条目。
6. 页面渲染元信息和媒体卡片。
7. 预览和下载都通过 `/api/media?url=...` 完成，这样媒体保持同源。

## 设计约束

- 不使用数据库
- 不提供登录流程
- 不保留服务端历史
- 不运行后台任务
- 每次交互只发起一次抓取请求
- 只有在展示或保存媒体时才走同源代理
